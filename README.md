# TOKKI

**TOEIC Optimized Knowledge & Keyword Index**  
약자는 **TOKKI**이고, 한글로는 **토끼**라고 읽어요.

Spring Boot React MySQL Gradle TypeScript Tailwind CSS Axios JavaScript

시중 TOEIC·영단어 앱을 보면 플래시카드에 **객관식·듣기·받아쓰기**를 붙이거나, 잊을 때쯤 다시 보여 주는 **간격 반복(SRS)** 으로 오답을 돌려주는 식이 많은데, 그런 방식도 암기에는 분명 도움이 돼요. 다만 시험장에 가면 결국 **스스로 떠올려서 써야** 하잖아요, 그래서 저희는 그 순간을 조금 앞당겨 보자는 쪽으로 방향을 잡았어요. 

토끼는 멀리를 한 번에 날아가지 않고, **짧게 여러 번 뛰면서** 길을 옮겨 가잖아요. 덩치 큰 도약이 아니라, 익숙해질 때까지 **작은 점프를 이어 가는 쪽**이 토끼다운 모습이에요. 그 모습을 보면서 TOEIC 단어도 **한 꺼풀씩, 오늘은 여기까지만**처럼 나눠 가지면 어떨까 생각했어요. 

그래서 점수대에 맞는 난이도를 세우고, 그 안을 또 **10단계**로 잘라 부담을 덜어 주고, 카드로 익힌 뒤 주관식으로 확인까지 이어지게 만든 게 바로 **TOKKI**예요. 

점수대마다 하·중·상으로 나눈 건 그대로 두었고, **영단어 탭**에서는 단어·뜻·예문을 카드로 익히고 **퀴즈 탭**에서는 시험에 가까운 느낌으로 넘어가게 흐름을 나눠 두었어요. 퀴즈는 전부 **주관식**이라, 보기만 고르는 객관식보다 **직접 쳐서 끄집어내는 연습**에 가깝게 쓰실 수 있어요.

문제를 다 풀고 나면 **문항마다 맞았는지** 바로 확인할 수 있게 해두었고, 틀린 단어는 **오답 노트**에 자동으로 쌓여요. 나중에 다시 맞혔을 때 노트에서 지울지 말지는 **본인이 선택**하실 수 있어요. 앱을 새로 깔 필요 없이 **브라우저**에서 열 수 있게 해두었고, 로그인은 평소 쓰시는 **구글 계정**으로 바로 들어오시면 돼요. 앞으로는 친구랑 같은 문제를 두고 붙어 보는 **실시간 1:1** 같은 모드도 더해져서, 혼자 외우다 지칠 때 동기가 생기도록 만들어 갈게요.

## 지금 버전에서 하는 일

- **로그인** — 평소 쓰시던 **구글 계정**으로 들어가실 수 있게 해두었어요. 이 사이트만의 비밀번호를 새로 만드시거나, 서버에 따로 저장해 두진 않아요.
- **난이도** — 하(500점대 이하)·중(500~800)·상(800 이상)으로 나누고, 각 난이도마다 **10단계**로 잘라서 진행하도록 해두었어요.
- **영단어 탭** — 단어·뜻·예문을 **카드**로 넘겨 가며 보실 수 있게 해두었어요.
- **퀴즈 탭** — **한→영 / 영→한**만 고르시게 해두었고, 문항은 전부 **주관식**으로 풀 수 있어요.
- **결과·오답** — 제출하시면 **문항마다 맞았는지** 보여 드리고, 틀린 건 **오답 노트**에 쌓이게 해두었어요. 다시 맞히셨을 때 노트에서 지울지는 **직접 선택**하실 수 있어요.
- **지난 결과** — 예전에 풀었던 세트는 다시 들어오셔서 **이전 결과**를 보실 수 있게 해두었어요.
- **운영** — 단어를 **하나씩 등록·수정·삭제**하실 수 있는 화면은 운영용 계정에만 열어두었어요.
- **화면** — 폰 브라우저에서도 레이아웃이 무너지지 않게 **반응형**으로 맞춰 두었고, 한 손으로 잡았을 때 **엄지**가 닿기 쉬운 쪽으로 버튼을 배치해 두었어요.
- **채점** — **대소문자**와 **앞뒤 공백**은 채점할 때 무시하도록 해두었어요.
- **콘텐츠** — 단어·뜻·예문은 공신력 있는 **사전 계열 출처**를 기준으로 다듬는 방향으로 정리해 두었어요.

## 로드맵 · 추가로 붙일 기능

아직 없는 기능들은 순서가 바뀔 수 있지만, 대략 이런 줄로 이어갈 예정이에요.

- **퀴즈 이어풀기** — 중간에 나가도 **문항 단위로 저장**해 두었다가 다음에 이어서 풀기
- **랭킹** — 전체 기준으로 자주 틀린 단어 **상위 목록**, **매일 자정**쯤 갱신
- **PvP(1:1)** — 둘이 같은 조건에서 **실시간 속도 대결**. 한쪽이 **방을 만들고 초대 코드**를 넘기면 상대가 입장, 시작 후에는 **상대 진행**이 화면에 보이게, 끝나면 **점수·시간**을 나란히 비교
- **발음(TTS)** · **연관 단어**(유의어·반의어·동의어·파생어 등) · **구문·숙어** · 상위 난이도 마지막 단계용 **비즈니스 예문**
- **오프라인** — 네트워크가 끊겼을 때를 대비해, 진행 상태를 **기기에 잠깐만** 저장해 두는 처리

## 로컬 Google OAuth2 테스트

이슈 #39의 실제 로그인 왕복 테스트는 DB 사용자 등록 없이 `auth-test` 프로필로 확인할 수 있어요.

1. Google Cloud Console의 승인된 리다이렉트 URI에 `http://localhost:8080/login/oauth2/code/google`를 등록합니다.
2. `.env`에 `GOOGLE_CLIENT_ID`, `GOOGLE_CLIENT_SECRET`, `TOKKI_FRONTEND_CALLBACK_URL`을 설정합니다.
3. 백엔드를 Java 21로 실행합니다. 프론트 dev 서버 없이 백엔드 테스트 페이지로 확인하려면 콜백 URL을 `http://localhost:8080/auth-test/callback`로 둡니다.

```powershell
$env:JAVA_HOME='C:\Program Files\Eclipse Adoptium\jdk-21.0.11.10-hotspot'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
cd backend
.\gradlew.bat bootRun --args='--spring.profiles.active=auth-test'
```

4. 브라우저에서 `http://localhost:8080/auth-test/login`에 접속해 Google 로그인 버튼을 누릅니다.

성공하면 `/auth-test/callback` 화면에서 “Google OAuth2 로그인과 서버 세션 확인에 성공했습니다.”가 표시됩니다. 서버 세션 확인은 `GET /api/auth/me`, 로그아웃은 `POST /api/auth/logout`, 최근 로그인 이벤트 확인은 관리자 키를 포함한 `GET /api/auth/events`로 확인할 수 있습니다.
